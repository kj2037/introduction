【メモ】
コミットは1作業ごとに行う。
コミットする変更内容はステージングエリアで選択する。
コミットはスナップショットを取ることで行われる
コメントは端的に分かりやすく書く。
ワークツリーは作業場と作業道具の意、ツリーがファイルを指す。
エイリアスは別名の意味、ユーザー辞書のイメージ
コマンドにエイリアスをつけておくと入力が楽になる

【ディレクトリのgit化手順】
1.mkdir ｘｘｘｘｘ で空のディレクトリ作成
2.cd ｘｘｘｘｘ で作成したディレクトリに移動
3.git init でディレクトリを .git フォルダ作成
4.ls -a で.git ファイル(隠れファイル)ができているか確認
5.ls .git/ でファイル内確認
6.config  description  HEAD  hooks/  info/  objects/  refs/ ができているか確認


【他人のプロジェクトをローカルにコピーする】
（git リポジトリのコピーを作成する。）
1．github で対象のリポジトリを検索
2. リンクをコピー
3. cd でコピーを置きたいディレクトリに移動
4, git clone で対象をコピー


【変更をコミットする】
1. git add . で全てのファイルをステージに追加
2. git commit でステージの内容をスナップショットとしてリポジトリに保存


【コマンド一覧】
gitの管理から外したいファイルは.gitignoreにまとめる

cat ~/ .gitconfig

mkdir
    ディレクトリ作成
rm
    ファイル削除
cp
    ファイルコピー
mv
    ファイル移動、ファイル名変更
cat
    空ファイル作成

cat ファイル名
    ファイルの内容を表示

touch ファイル名

ls
    ディレクトリ内のファイルを表示
ls -a
    ディレクトリ内のファイルを隠しファイルも含めて表示

git init
    ローカルリポジトリ作成(blob, tree, commit, index など)

git diff
    ステージングエリアとローカルファイルの変更差分を表示
    "q"が終了コード
git diff ファイル名
    ステージングエリアとローカルファイルの変更差分を表示
git diff --staged
    ステージングエリアと最新コミットの差分

git add xxxxxxx.yyy
    指定したファイルの変更をステージングエリアへ登録する
git add .
    全ての変更をステージングエリアへ登録する

git rm ファイル名
    ステージングエリアからファイルを削除
git rm --cached ファイル名
    対象をgitの記録(リポジトリ)から削除するがローカル(ワークツリー)にファイルは残す
git rm -r ディレクトリ名
    ディレクトリを削除

ci
git commit -m "コメントを書く"
    ローカルリポジトリへコミット
git commit -v
    変更内容を確認してコミット
git commit --amend
    直前のコミットをやり直す。ただしpushしたコミットについては使用しないこと。

st
git status
    リポジトリ内の状態確認

git log
    コミットログの表示
git log --oneline
    コミットログの一行で表示
git log -p xxxxxxx.yyy 
    ファイルの差分を表示する(qで操作終了)
git log -n 3 
    最新3つのコミットログ表示

git mv 旧ファイル 新ファイル
    ファイル名を変更する。

git remote 
    リモートリポジトリの名前を表示
git remote -v
    リモートリポジトリの名前,URLを表示
git remote add xxxx https://~~~~~~~~~~~~~~~
    xxxxというショートカットでurlのリモートリポジトリを登録する.
git push -u リモート名 ブランチ名
    -u をつけることで次回以降のpushでリモート名、ブランチ名を省略できる
git push リモート名 ブランチ名
    ローカルリポジトリの内容をリモートリポジトリ(共有リポジトリ)にpush

git clone https*//~~~~~~~~~~~~~~~~~~~~~~
    指定したリモートリポジトリをローカルにコピー

git checkout --ファイル名
    ファイルの変更を取り消す。ワークツリーの状態をステージと同一にする。

git reset HEAD ファイル名
    ステージの変更を取り消す。

git fetch リモート名
    リモートの情報をローカルにコピー
git merge リモート名/ブランチ名
    fetch後のローカルリポジトリの内容をワークツリーに反映させる。
git pull リモート名 ブランチ名
    fetch, merge を同時に行う。


rm
    ファイルの削除
